­­­­­Parameters

  • mu - Population Size (# of test suites) – likely small ~10 to 50
  • initial Test Suite Size
  • This should probably be a function of the complexity of the control flow graph
  • maximum number of test cases allowed in a test suite
  • This should probably also be a function of the complexity of the control flow graph
  • How many cut points to use in test suite crossover
  • How many cut points to use in test case crossover
  • Crossover probability for test suites
  • Crossover probability for test cases

Algorithm

  1. Initialization
  1. Generate mu test suites of size initialTestSuiteSize randomly and add them to the population
  2. Evaluate the fitness of each test suite
  1. Evaluate the paths covered by each of the test cases and map each test case to a path (sequence of conditions in the CFG)
  2. Determine fitness based upon a measure of how many paths out of total paths are covered. And some other factors.
  1. While(stopping criteria has not been met)
  1. Choose two parents using fitness proportional selection
  2. Apply test suite level crossover to generate two offspring test suites
  3. Foreach offspring test suite
  1. Foreach test case in an offspring
  1. Flip a coin with probability SelectTCProb
  2. If heads,
  1. Add to pool of test cases to apply crossover and mutation to
  1. For each pair of test cases in the pool
  1. Apply test case level crossover
  1. Crossover entire parameters. Need to decide how we are deciding number of cut points
  1. Apply parameter level mutation to both offspring
  2. Foreach parameter
  1. Flip a coin with probability ParamMutProb
  2. If heads, mutate the parameter
  1. Evaluate the paths of both offspring test cases. Keep them in the test suite iff it covers a previously uncovered path
  1. Evaluate the fitness of each offspring test suite
  2. Replace one or both of the parent test suites if improvement has been made

Stopping Condition

  1. A test suite is found that covers all paths
  2. A test suite can be built from existing test cases that covers all paths
  3. A large number of generations
  4. A certain number of generations passes with no substantial increases in fitness
